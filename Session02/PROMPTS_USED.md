# Prompt Engineering Workflow – Wanderfolk

This project was developed using iterative prompt engineering. Instead of generating the complete application in one prompt, the project was built incrementally by refining prompts throughout the design, development, and deployment process.

---

# Prompt 1 – UI Concept Generation

**Objective:** Generate a modern, aesthetic travel planner UI.

```
Design a premium AI-powered travel planner website inspired by editorial travel magazines and modern booking platforms.

The website should have a warm, beige and sandy color palette with minimalist typography and smooth animations.

The landing page should include:

- Hero section
- Popular destinations
- Travel journal
- How it works
- AI Trip Planner
- Saved trips
- Footer

Focus on creating a luxurious yet calm travel experience rather than a traditional booking website.
```

---

# Prompt 2 – Hero Animation

**Objective:** Create a memorable first impression.

```
Design a cinematic hero section.

When the website loads, a girl skateboards through a desert landscape while sand particles subtly move in the background.

As she finishes skating, the website title smoothly fades into view.

The animation should feel elegant and realistic rather than cartoon-like.

Avoid using a static cutout moving left to right.

Use parallax effects, subtle dust movement, and cinematic easing.
```

---

# Prompt 3 – Pixel Perfect Implementation

**Objective:** Convert the UI into production-ready frontend.

```
Implement the UI exactly as designed.

Requirements:

• Pixel-perfect implementation
• Fully responsive
• Mobile-first
• Semantic HTML
• Accessible components
• Clean folder structure
• Reusable React components
• Smooth hover animations
• Optimized performance
```

---

# Prompt 4 – AI Integration

**Objective:** Connect backend functionality.

```
Integrate all required APIs into the application.

Generate personalized itineraries using the AI model.

Include:

• Destination
• Budget
• Duration
• Travel preferences

Display:

• Day-wise itinerary
• Estimated expenses
• Weather
• Travel recommendations

Handle loading states and API errors gracefully.
```

---

# Prompt 5 – Navigation & UX Improvements

**Objective:** Improve usability.

```
Enhance the user experience without changing the design.

Implement:

• Smooth scrolling
• Active navbar highlighting
• Responsive navigation
• Better user flow
• Functional buttons
• Improved accessibility

Every visible interactive element should perform a meaningful action.
```

---

# Prompt 6 – Destination Details

**Objective:** Make destination cards useful.

```
When a user clicks "Explore More," open a destination detail page.

Include:

• Overview
• Best season
• Attractions
• Local cuisine
• Activities
• Travel tips
• Budget
• Transportation

Add a "Plan This Trip" button that automatically opens the AI planner with the selected destination prefilled.
```

---

# Prompt 7 – Journal Experience

**Objective:** Make travel stories interactive.

```
Each journal card should open a dedicated article page.

Include:

• Hero image
• Reading time
• Travel story
• Hidden gems
• Local recommendations

At the end of every article include a CTA:

"Plan This Trip"

which opens the AI planner with the destination already selected.
```

---

# Prompt 8 – Saved Trips

**Objective:** Improve trip management.

```
Create a Saved Trips dashboard.

Users should be able to:

• View saved trips
• Edit
• Delete
• Duplicate
• Reopen itineraries

Persist data using local storage.

Display destination, duration, budget, weather summary, and thumbnail.
```

---

# Prompt 9 – Dynamic Images

**Objective:** Replace placeholder images.

```
Replace placeholder destination images with dynamically fetched images.

Retrieve destination-specific images using a free image API.

Populate galleries with:

• Landmarks
• Culture
• Activities
• Nature
• Food
• Local people

Fallback to a placeholder image if no result is found.
```

---

# Prompt 10 – Production Polish

**Objective:** Final optimization before deployment.

```
Prepare the application for production.

Ensure:

• Responsive layouts
• Performance optimization
• Error handling
• Loading states
• Accessibility
• Smooth animations
• Component reusability
• Clean code organization

Do not redesign the UI.

Focus on functionality, reliability, and user experience.
```

- ✅ Structured Prompting

# Development Strategy

Instead of generating the complete application in a single prompt, the project followed an iterative workflow:

1. UI ideation
2. Hero animation
3. Pixel-perfect frontend implementation
4. AI integration
5. UX improvements
6. Navigation refinement
7. Feature implementation
8. Performance optimization
9. Production deployment

This iterative prompting approach allowed continuous refinement of both functionality and user experience while maintaining consistency throughout the project.
